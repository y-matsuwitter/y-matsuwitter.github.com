Pathが全アドレスブックデータをサーバーにあげてたらしい。

それを発見した人のブログ記事を英語の勉強がてら超意訳してみた。

誤訳御免。

#

始まりはふとしたことだったんだ。

定例のハッカソンで僕はPathの実装について考えていた。

そこで先日Hacker Newsを騒がせた素晴らしいmitmproxyってツールでPathのiPhoneからサーバーへの通信を観察してみた。

これらは全く無害なものだと思っていたんだ。

https://api.path.com/3/contacts/addへのPOST リクエストを覗くまではね。

よくよく見てみると、僕のアドレス帳の全データがplist形式でPathへ送られていることが分かったんだ。

Pathに対してアドレス帳のアクセス許可やそのサーバーへの送信を許可した記憶なんて無かったから、とりあえずもう一度新しくPathをインストールして実験してみた。

そしてその結果は全く同じだった。つまり僕のアドレス帳の全データがサーバーへ送られていたんだ。

注意

僕はPathが僕のアドレス帳データを使って何かひどいことをしているなんて全く思ってないよ。

ただサードパーティー製のアプリにアドレス帳のデータが送られるような事実が怖い。

僕はiOSアプリとしてPathを愛しているし素晴らしいエンジニアがそこで働いているんだと思う。

ただ無意識にアドレス帳が送信されるという事実だけはちょっと怖い。

他のiOSアプリが同じことをやっているかもしれないんだから・・・

追記

PathのCEODave Morinからコメントで返信があったので掲載しておく。

我々はこの問題に関して慎重に議論すべきだと思う。

余談だけど、こんな風に返信がもらえるなんて全く考えもしなかった。

Aran、指摘ありがとう。

これが非常に大事な議論であり、真剣に行うべきだと思う。

我々はユーザーに友達を見つけて繋がってもらい、Pathに早く親しんでもらうために、そして友達や家族がPathに参加したときにお知らせするためにアドレス帳をサーバーへアップロードしていたんだ。それ以上の他意はない。

我々はこう行った友達の発見とマッチングがこの産業にとって重要でありユーザーがそのことを理解していることも重要だと信じている。故に我々は率先してAndroidクライアント向けのアドレス帳収集のオプトインを数週前に実装し、iOSアプリのver2.0.6でも実装しAppStoreの許可を待っている。

Dave Morin

Co-Founder and CEO of Path
